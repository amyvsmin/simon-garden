# 飛飛（林子婷）— 為什麼資安意識總是被遺忘？人因 + 故事化的有感訓練設計｜CYBERSEC 2026 Cyber Talent Day 3

## 摘要

七維思執行長飛飛（林子婷）十年資安意識訓練經驗的總結：完訓率 95–100% 卻換不到行為改變、釣魚演練點選率持續居高、根因是訓練缺乏情緒與故事——大腦只記心跳加速的瞬間跟有情節的故事、條列式注意事項記不住。她拆出兩個核心問題：**效果問題**（員工聽完就忘、回去還是會點）跟**規模化問題**（只有少數人會寫情境、年年從零開始）。對應方案是「有感訓練四元素」：情境故事 + 角色視角 + 情緒鉤子 + 決策後果，加上「拼積木框架」把訓練情境拆成「角色卡 × 攻擊手法卡 × 場景卡」三維組合、讓不同部門、年份、攻擊面都能五分鐘產出客製情境。社交工程被解析成四大套路：**權威冒充**（IT/主管/政府冒名→不敢質疑）、**緊急施壓**（時間壓力→跳過驗證）、**利益誘惑**（退稅／中獎→引誘確認）、**好奇陷阱**（模糊警告→引發點選）。改良式釣魚演練流程從「考點選率」升級為「練通報率」：演練前用故事鉤子預告、演練中即時反饋還原決策心理、演練後做覆盤討論而不是責備、一週後追蹤通報率作為行為改變指標。日常口訣三字：**換**（換管道確認）、**等**（等一下再做）、**問**（問信任的人）。

## 核心概念

- [[security-awareness-design]]：有感訓練四元素（情境故事 + 角色視角 + 情緒鉤子 + 決策後果）；行為改變才是目標、完訓率不是
- [[social-engineering-playbook]]：四大攻擊套路（權威冒充 / 緊急施壓 / 利益誘惑 / 好奇陷阱）；每種套路三段式公式
- [[phishing-drill-improved]]：通報率取代點選率為核心指標；演練前/中/後三階段設計；引導式覆盤代替責備
- [[storytelling-modular-framework]]：角色卡 × 手法卡 × 場景卡三維組合、5 分鐘產情境、解規模化問題

## 對 Simon 的應用

公司 IT + 個人寫作雙軸的應用：

1. **公司資安訓練重做**：目前每月給同仁的資安訓練偏條列、可改用飛飛的四元素重寫一次釣魚演練流程：
   - 演練前一天發「公司過去半年釣魚事件回顧」當預告（好奇心啟動）
   - 演練信加客製職位場景（給財務的用發票、給業務的用客戶訂單）
   - 點選後不直接責備、用引導頁問「為什麼會點？」
   - 一週後給管理層看通報率提升曲線、不是點選率下降
2. **「換等問」三字口訣可張貼**：印小貼紙貼螢幕邊、Sam 月會也能順便推、零成本高 ROI
3. **拼積木框架可搬到 ISO 27001 教育訓練**：本來每年都重寫資安培訓投影片、改用角色卡（行政/業務/IT/工程師）× 手法卡 × 場景卡組合、可重複用且每年只更新少數場景卡
4. **個人寫作借鑑**：Substack 寫資安/AI 文章時、用「情境 + 角色 + 情緒鉤子 + 後果」四元素、避免條列式說教、讓讀者真的記住
5. **AI 直接生情境**：把飛飛的角色卡/手法卡格式餵給 Claude、一分鐘產出 10 個訓練情境、解決「找不到人寫」的規模化痛點

## 原文要點

### 訓練失效的根因

- 完訓率 95–100% 但點選率沒降→沒改變行為
- 大腦只記情緒+故事、條列式記不住
- 釣魚演練只考「點不點」、不問「為什麼會點」

### 有感訓練四元素

1. **情境故事**：「週五下午五點」這種具體場景、員工覺得「這會發生在我身上」
2. **角色與視角**：業務、新人、財務各有自己的壓力源；訓練要寫不同 persona
3. **情緒鉤子**：核心是讓員工「真的猶豫」、猶豫代表判斷系統啟動了
4. **決策與後果**：每個選項都有後果、做錯的後果才是學習錨點

### 社交工程四大套路

| 套路 | 三段式公式 | 範例 |
|---|---|---|
| 權威冒充 | 身份確立 → 要求行動 → 暗示後果 | 「我是 IT 主管、你應該照做、否則賬號停用」 |
| 緊急施壓 | 緊急詞 + 後果 + 行動指令 | 「下班前一定要、否則出問題、馬上點連結」 |
| 利益誘惑 | 好消息 + 限時行動 + 連結 | 「退稅已核准、限時、登入確認賬戶」 |
| 好奇陷阱 | 模糊警告 + 好奇細節 + 行動指令 | 「有人標記了你的照片、看是不是你、點這裡」 |

### 拼積木框架（解規模化問題）

- **公式**：角色 × 手法 × 場景 = 訓練情境
- **角色卡**：8 種以上 persona（新人/主管/業務/財務/IT/助理/遠端員工/資深員工），每張寫動機、壓力點、可被利用的心理
- **手法卡**：四大套路（權威冒充/緊急施壓/利益誘惑/好奇陷阱）
- **場景卡**：時間（週五下班前、假日晚上）、管道（Email/Line/Teams/電話）、背景（趕案子、家庭壓力）
- **組合舉例**：助理 + 權威冒充 + 週五晚上 Line 訊息 = 假老闆要求買禮品卡
- **AI 賦能**：把卡片格式餵 Claude / ChatGPT、5 分鐘產訓練情境

### 改良式釣魚演練流程

1. **演練前**：發微劇情預告、用故事鉤子引發好奇心、不是制式通知
2. **演練中**：點擊後 landing page 問「這封信為何讓你想點？」、附簡短解說、不直接責備
3. **演練後**：引導式覆盤、員工還原決策過程「我當時是這樣想的」、理解而非檢討
4. **一週後**：追蹤通報率（不是點選率）作為行為改變指標

### 三字口訣：換等問

- **換**：換管道確認（Email 來的訊息打電話確認）
- **等**：等一下再做（給自己緩衝時間思考、攻擊者就靠你不思考）
- **問**：問信任的人（同事/主管/IT、通報是保護團隊）
