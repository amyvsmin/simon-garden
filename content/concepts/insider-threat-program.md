# Insider Threat Program

## 定義

組織針對內部威脅的制度化防禦框架。美國總統 Executive Order 要求關鍵基礎設施組織必建置；美國中大型企業近年也在原 incident response 之外另設獨立 insider threat mitigation team、降低約 50% 事件量。

## 關鍵面向

- **五大要點（陳浩維演講）**：
  - **Crown jewel 盤點**：找出組織最重要的資產（資料、系統、IP）
  - **跨部門整合**：資安／法務／HR 協作、不只是資安部的事
  - **通報機制 + SOP**：建立可疑行為的明確通報路徑
  - **技術偵測**：UEBA、DLP、Endpoint 異常分析
  - **資安文化**：全員參與、員工願意主動回報
- **參考資源**：CISA Insider Threat Library 含詳細範本與導入步驟
- **典型團隊組成**：資安工程師 + 數位鑑識 + HR liaison + 法務 + 通報窗口

## 應用場景

- Simon 工作場景：公司 ISO 27001 推進可把這 5 要點當章節範本、特別是 crown jewel 盤點銜接既有資產盤點階段
- 一般場景：金融、醫療、關鍵基礎設施、跨國電商；只要有長壽命敏感資料就值得做
